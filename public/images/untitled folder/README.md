# Homepage images (local only)

All photography and textures for the homepage live under `public/images/` with **semantic filenames** matching the editorial plan (hero, explore, neighborhoods, editorial, CTA, lifestyle, social, textures).

## Regenerating procedural placeholders

The repo ships **local** WebP/JPEG/PNG files generated by:

```bash
npm run gen:images
```

That runs `scripts/generate-local-editorial-assets.mjs` (Sharp) to write warm, editorial-toned gradient plates so every path referenced in `lib/home-content.ts` and `lib/site.ts` exists. **Replace these files** with your final photography when ready—keep the same paths and names so the layout does not change.

## Performance

- Hero and large bands use **WebP** at modest dimensions suitable for `next/image` `sizes` in the components.
- **OG** image: `editorial/og-share-1200.webp` (1200×630).

## External URLs

There are **no** Unsplash/Pexels or other remote image URLs in the app source. The Instagram field in `lib/site.ts` is a **profile link**, not an image `src`.
